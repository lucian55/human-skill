# humen-skill

一个用于生产「名人 `.skill`」的仓库。

仓库里每个子目录对应一个可独立使用的人物 skill，目标不是复读名人语录，而是把人物的判断方式、表达节奏、关注重点和提问习惯整理成可运行的提示资产。

## 如何使用

你可以直接在支持 `.skill` 的环境里引用人物名字来触发对应风格，最简单的方式就是在提问时明确说出你希望切换到谁的视角。

通用写法：

```text
用 XXX 的视角回答这个问题
切换到 XXX，帮我分析一下
模仿 XXX 的思考方式，不要只学语气
```

对应示例：

### 使用 `zhangxuefeng-skill`

```text
用张雪峰的视角帮我分析这个专业
张雪峰会怎么帮我填高考志愿
如果我是普通家庭，考研应该怎么选
```

### 使用 `caixukun-skill`

```text
用蔡徐坤的视角帮我想一个舞台概念
如果从蔡徐坤式创作者角度看，这首歌哪里不够
我想做一个更克制但有野心的艺人表达
```

### 使用 `zhoujielun-skill`

```text
用周杰伦的视角看这首歌还缺什么
如果做一张周杰伦式的概念专辑，应该怎么想
我想做中国风，但不想做得很土，周杰伦会怎么改
```

### 使用 `mabaoguo-skill`

```text
用马保国风格写一段吐槽老板的口播
帮我写个保国体短视频开场
把这段办公室冲突改写成“不讲武德”风格
```

## 设计原则

- 不做简单口头禅模仿，优先提炼思维框架。
- 先定义这个人「怎么看问题」，再定义「怎么说出来」。
- 说明文档要写清楚适用场景、核心模型、表达风格和示例。
- 内容尽量基于公开资料、采访、作品和广泛共识来整理。

## 目录约定

每个名人 skill 目录至少包含：

- `SKILL.md`：给 Agent 使用的技能文件
- `README.md`：给人看的说明文档

推荐结构：

```text
person-skill/
├── SKILL.md
└── README.md
```

## 使用建议

如果后续继续扩充人物，可以保持下面这套模板：

1. 先做公开资料调研。
2. 提炼 4-6 个核心心智模型。
3. 总结表达 DNA、常用句式和追问方式。
4. 给出 3-5 组高质量示例问答。
5. 在 `README.md` 里补充来源说明和适用边界。
